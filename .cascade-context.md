```sql
CREATE TABLE processing_jobs (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID NOT NULL REFERENCES auth.users(id),
  account_id UUID NOT NULL,
  file_url TEXT NOT NULL,
  status VARCHAR(20) DEFAULT 'pending',
  transactions_extracted INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 🔗 NEW n8n Workflow Design

### Workflow Structure (SEPARATE FROM EXISTING)
```
Webhook Trigger
├── Validate Request
├── Update Job Status: "processing"  
├── Download File from Supabase Storage
├── Get User Context (categories, existing transactions)
├── AI Processing (OpenAI/Claude with user context)
├── Category Mapping (user's categories)
├── Deduplication (against user's data)
├── Save Transactions to Supabase
├── Update Account Balances
└── Update Job Status: "completed"
```

### Key Differences from Existing Workflow
- ❌ NO Google Drive trigger
- ❌ NO Google Sheets integration
- ✅ Webhook-based processing
- ✅ Direct Supabase integration  
- ✅ User context awareness
- ✅ Real-time status updates
